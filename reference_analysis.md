# [프로젝트 제안서] 다중 모델 기반 수요 예측 및 물류 비용 최적화 시스템 구축

## 1. 개요 (Introduction)
본 프로젝트는 브라질 이커머스(Olist) 데이터를 활용하여 제품 카테고리별 수요를 다각도로 예측하고, 이를 경영과학의 최적화 모델과 결합하여 기업의 물류 운영 비용을 최소화하는 **'지능형 의사결정 지원 시스템'** 구축을 목표로 합니다. 어디서든 수요와 공급이라는 기본적인 구조는 존재하기 때문에, 의료에서도 충분히 활용 가능하다고 판단했습니다.

## 2. 배경 및 필요성 (Background)
* **물류 불확실성 증가:** 글로벌 이커머스 시장의 확대에 따라 배송 거리, 무게, 수요 변동성에 따른 물류비용 관리가 기업 경쟁력의 핵심으로 부상했습니다.
* **통합적 의사결정 결여:** 기존 시스템은 단순 예측(Forecasting)에 그쳐, 실제 비즈니스 제약 조건을 고려한 최적 발주 및 배송 전략(Optimization)으로 이어지지 못하는 한계가 있습니다.
* **데이터 사이언스의 실무 적용:** 시계열 분석의 다양한 방법론을 실제 비즈니스 데이터에 적용하여 실무적인 인사이트를 도출하고자 합니다.

## 3. 목표 (Objectives)
* **다중 모델 비교 분석:** ARIMA,  랜덤포레스트, XGBoost, LSTM 세 가지 모델을 구현하여 데이터 특성에 따른 최적의 수요 예측 알고리즘을 규명합니다.
* **비용 최소화 알고리즘 구현:** 예측된 수요를 기반으로 선형계획법(LP)을 적용하여 총 물류 비용을 최소화하는 최적 해를 도출합니다.
* **확장 가능한 아키텍처 설계:** OpenClaw 구조를 벤치마킹하여 AI 에이전트가 코드를 파싱하고 실행하기 용이한 모듈형 시스템을 구축합니다.

## 4. 기존 프로젝트 대비 개선점 (Improvements)
* **단일 모델 → 다중 모델 앙상블:** 단순 LSTM 사용에서 벗어나 통계적 모델(ARIMA)과 머신러닝(XGBoost)을 추가하여 분석의 객관성을 확보했습니다.
* **단순 예측 → 의사결정 최적화:** 예측 정확도($SMAPE$) 향상에만 집중하던 기존 방식에 경영과학의 '최적화' 단계를 추가하여 비즈니스 가치를 창출합니다.
* **전통적 구조 → 에이전트 친화 구조:** 사람이 읽기 위한 코드를 넘어, 독립적인 에이전트가 플러그인처럼 작동할 수 있도록 결합도를 낮춘 아키텍처를 적용합니다.

## 5. 데이터셋 및 분석 (Dataset)
* **출처:** Kaggle - Brazilian E-Commerce Public Dataset by Olist (약 10만 건)
* **전처리 전략:** - **Python 전용 파이프라인:** Pandas를 활용하여 9개의 관계형 CSV 테이블을 통합 관리합니다.
    - **피처 엔지니어링:** Lag feature, Rolling window, 지역별 운송비 가중치 등 시계열 특화 피처를 생성합니다.
* **평가 지표:** $SMAPE$ (수요 예측), $Total\,Logistics\,Cost$ (최적화 평가)

## 6. 주요 기능 (Core Functions)
* **Multi-Model Forecaster:** ARIMA(통계), 랜덤포레스트, XGBoost(ML), LSTM(DL) 기반의 수요 예측 에이전트 모듈.
* **Inventory Optimizer:** 예측 데이터를 상수화하여 재고 유지비와 발주 비용을 최소화하는 수리 모델링 엔진.
* **Agent-Friendly Architecture:** OpenClaw 아키텍처를 참고하여 `src/agents/` 및 `src/prompts/`로 분리된 모듈형 구조.

## 7. 기대 효과 (Expected Effects)
* **비용 절감:** 데이터 기반 최적 발주를 통해 불필요한 재고 비용 및 품절 손실을 최소화합니다.
* **분석 생산성 향상:** 에이전트 친화적 구조를 통해 향후 분석 자동화 및 모델 업데이트가 용이합니다.
* **도메인 융합 역량:** 데이터 사이언스와 경영과학 기법을 결합한 엔드투엔드 문제 해결 능력을 입증합니다.

## 8. 향후 일정 (Project Roadmap)
* **1~2주 차:** 원본 데이터 분석(EDA) 및 Pandas 기반 데이터 통합 파이프라인 구축
* **3~4주 차:** ARIMA, XGBoost, LSTM 모델 구현 및 하이퍼파라미터 튜닝(Optuna 활용)
* **5~6주 차:** 선형계획법(LP) 기반 비용 최적화 엔진 구축 및 모델 결합
* **7~8주 차:** 최종 성과 지표 분석, 성능 비교 리포트 작성 및 레포지토리 정리
