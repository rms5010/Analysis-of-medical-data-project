# 📦 이커머스 물류 최적화를 위한 수요 예측 및 비용 분석 시스템

---

## 1. 참고 프로젝트 개요

본 프로젝트는 Kaggle의 *Brazilian E-Commerce Public Dataset by Olist* 데이터를 활용하여, 이커머스 환경에서의 주문, 상품, 배송 정보를 분석하는 것을 목표로 한다.

해당 데이터는 실제 이커머스 플랫폼의 주문 데이터를 기반으로 구성되어 있으며, 고객 위치, 상품 특성, 배송 정보 등을 포함하고 있어 다양한 관점에서 분석이 가능하다.

특히 본 프로젝트에서는 단순 데이터 분석을 넘어, 수요 예측과 물류 비용 분석을 결합하여 데이터 기반 의사결정 가능성을 탐색하고자 한다.

---

## 2. 선택 이유

기존의 Store Item Demand Forecasting Challenge와 같은 수요 예측 문제를 참고하여, 이를 보다 현실적인 이커머스 데이터에 적용해보고자 본 프로젝트를 기획하였다.

또한 GitHub에 공개된 데이터 분석 프로젝트를 참고하여 전체적인 분석 흐름을 이해하였으며, 본 프로젝트에서는 사용 데이터와 분석 목적을 변경하여 보다 현실적인 비즈니스 문제로 확장하였다.

특히 기존 프로젝트가 주로 수요 예측에 초점을 맞춘 반면, 본 프로젝트에서는 예측 결과를 물류 비용 분석과 연결하여 의사결정 문제까지 확장하고자 한다.

---

## 3. 해당 프로젝트 분석 내용

### 3.1 데이터 특성 분석

* 다수의 CSV 파일로 구성된 관계형 데이터 구조
* 주문, 상품, 고객, 지역 정보 포함
* 배송비(freight_value)와 상품 특성(무게, 부피) 데이터 포함

👉 여러 데이터를 병합하여 분석용 데이터셋을 구성할 필요가 있음

---

### 3.2 데이터 전처리 및 피처 엔지니어링

* Pandas를 활용한 데이터 병합 및 정제
* 결측치 처리 및 불필요한 컬럼 제거
* 시간 정보 기반 피처 생성 (월, 요일 등)
* 카테고리 및 지역 단위 데이터 집계

---

### 3.3 모델링 분석

* 기본 모델: Linear Regression, Random Forest
* 추가 모델(선택): LSTM

기본 모델을 활용하여 안정적인 예측 결과를 확보하고, 추가적으로 LSTM을 적용하여 시계열 데이터의 특성을 반영한 성능 비교를 수행한다.

---

### 3.4 성능 평가 분석

* RMSE: 예측값과 실제값 간 오차 측정
* SMAPE: 비율 기반 오차 측정

또한 하이퍼파라미터 조정 및 모델 비교를 통해 성능 개선을 시도한다.

---

### 3.5 물류 비용 분석 (핵심)

본 프로젝트에서는 수요 예측 결과를 활용하여 물류 비용을 분석한다.

물류 비용은 다음과 같이 구성한다.

* 배송 비용: 데이터에 포함된 freight_value 활용
* 재고 유지 비용: 일정 비율을 가정하여 계산

이후 수요 예측을 활용한 경우와 활용하지 않은 경우를 비교하여, 총 물류 비용(배송비 + 재고 비용)의 변화를 분석한다.

이를 통해 수요 예측 기반 의사결정이 비용 절감에 미치는 영향을 평가한다.

---

## 4. 프로젝트 구조 분석

본 프로젝트는 데이터, 코드, 결과 파일을 폴더별로 나누어 정리하여 관리한다.

* data: 원본 및 전처리 데이터 저장
* src(code): 분석 및 모델링 코드
* output: 결과 및 시각화 자료

이를 통해 프로젝트의 가독성과 관리 효율성을 높이고, 각 기능을 독립적으로 수행할 수 있도록 구성한다.
